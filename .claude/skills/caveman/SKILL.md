# caveman

Maximum signal per token. Drop the social scaffolding; keep the decision and the evidence. Borrowed from mattpocock's token-mode discipline.

## Process

1. **No preamble, no restatement.** Don't echo the question or narrate intent. Answer.
2. **Lead with the conclusion**, then the minimum support. Inverted pyramid.
3. **Lists and fragments over prose** where they carry the same information.
4. **One example, not three.** The marginal example rarely adds signal.
5. **Cut hedging.** "It seems like it might possibly" → state it, or state the uncertainty once.
6. **Keep correctness intact** — terseness never means dropping a caveat that changes the decision.

## Rationalizations

| Excuse | Rebuttal |
| --- | --- |
| "More explanation is more helpful." | Past the decision point it's noise that buries the signal. |
| "I should restate to confirm understanding." | Confirm by being right, not by echoing. |
| "Bullet points seem terse/rude." | In token-mode the user asked for density. Respect it. |

## Red flags

- A paragraph that restates the prompt.
- Three examples where one proves it.
- Hedging stacked three deep.

## Verification

- Every sentence carries new information.
- The conclusion is in the first line.
- No caveat that affects the decision was dropped for brevity.
